# 错误处理与重试策略 (Error Handling)

---

## 错误分级

| 等级 | 场景 | 处理 |
|:-----|:-----|:-----|
| P0 | 关键字段缺失、代码读取全面失败、前序文件不存在、subagent 连续 blocker、校验 2 次失败 | 立即中止，报告错误与修复建议 |
| P1 | 部分代码未读取、选型信息不完整、reviewer 首轮发现可修复问题 | 记录警告到 `progress.md`，继续执行 |
| P2 | 格式细节偏差、非关键字段缺省值 | 仅日志记录，不中断 |

---

## 重试策略

### 校验脚本失败
1. 读取错误输出，定位问题
2. 定向修复
3. 重新校验
4. **最多 2 轮**，超出 → P0

### Subagent BLOCKED
1. 补充上下文重发（1 轮）
2. 仍 BLOCKED → 尝试拆分 Task
3. 拆分后仍 BLOCKED → P0，review plan.md

### 连续失败
- 连续 2 个 Task BLOCKED → P0，暂停 Phase 3

### Reviewer 驳回
1. 修复项传回 implementer
2. 修复后 re-review（最多 2 轮）
3. 超出 → P0
4. NEVER 跳过 re-review

### 代码检索失败
1. 替代检索策略
2. 2 次失败 → 降级，列出需用户提供的清单
